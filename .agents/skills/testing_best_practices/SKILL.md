# テスト ベストプラクティス スキル

## ベストプラクティス
- Terraform/Python/dbt でテスト責務を分離する。
- 変更差分に対して最小1件の回帰テストを追加する。
- CIで `terraform validate`, `dbt test`, `pytest` を実行する。
- 失敗時は再現手順と修正方針を記録する。

## 設計原則
- テストは仕様（Spec）から逆算して設計する。
- セキュリティ、データ品質、冪等性を優先して検証する。
- 本番相当データ量を想定した性能観点を含める。

## アンチパターン
- 実装後にテスト対象を限定して都合よく省略する。
- 期待値を曖昧にしたスナップショット依存テスト。
- CIでテスト失敗を許容したままマージする。
