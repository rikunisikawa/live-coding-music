# セキュリティ ベストプラクティス スキル

## ベストプラクティス
- IAM最小権限を徹底し、ロール分離を行う。
- 機密データは暗号化（保存時/転送時）を必須化する。
- CloudTrail/Config/GuardDuty 等の監査と検知を有効化する。
- PII/医療情報には分類タグとアクセス監査を適用する。

## 設計原則
- セキュリティは設計段階（Architect）で制御点を定義する。
- 実装前に脅威とデータ流通経路を明確化する。
- 監査証跡をテスト可能な形で残す。

## アンチパターン
- 長期アクセスキーの共有運用。
- 機密データを平文ログへ出力する。
- 本番データを開発環境へ無制限に複製する。
