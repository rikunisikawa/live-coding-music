# データモデリング スキル

## ベストプラクティス
- 各テーブルで Grain を明文化する。
- Factはイベント、Dimensionは属性として分離する。
- サロゲートキーは再現可能な方式で生成する。
- モデルごとに主キー/外部キー制約の検証を行う。

## 設計原則
- Lineageを `source -> staging -> core -> mart` で追跡可能にする。
- 指標定義は1箇所に集約し、重複計算を避ける。
- スキーマ変更は後方互換性を評価して反映する。

## アンチパターン
- 粒度の異なるデータを無条件に結合する。
- 1テーブルにFactとDimensionを混在させる。
- 履歴管理が必要な属性を上書き更新のみで扱う。
