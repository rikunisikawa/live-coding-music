# AWSアーキテクチャ スキル

## ベストプラクティス
- マルチアカウント戦略（dev/stg/prod分離）を前提にする。
- VPCは用途別サブネット（private優先）で構成する。
- S3は暗号化（SSE-KMS）、バージョニング、ライフサイクルを有効化する。
- Glue Data Catalog を単一メタデータ基盤として管理する。
- Redshift はWLM/ソートキー/ディストリビューション戦略を明示する。

## 設計原則
- 最小権限、監査可能性、再現性、可観測性を優先する。
- データ基盤は Raw/Core/Mart を明確に分離する。
- 医療データは機密区分ごとに保管・アクセス制御を分ける。

## アンチパターン
- 単一アカウントに本番と検証を混在させる。
- S3バケットを暗号化・監査ログなしで利用する。
- 手動コンソール設定を運用前提にする。
